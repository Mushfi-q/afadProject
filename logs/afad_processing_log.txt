AFAD DATASET PREPARATION LOG
==============================
Project: Artificial Familiarity Attack Detector (AFAD)
Date: 2026-03-14
Process: Clean, Merge, Filter, and Split Dataset for NLP Classification

1. PROCESS OVERVIEW
-------------------
The 'prepare_afad_dataset.py' script was refined to meet project standards:
- Data Source Integration: SMS Spam, Enron Email, and Synthetic Attack datasets.
- Text Preprocessing: Lowercase, URL removal, punctuation removal, whitespace normalization.
- Data Quality Filters:
    * Removed duplicates and empty records.
    * Added Length Filter: Only messages with 3 or more words are retained to provide sufficient context for NLP models.
- Class Balancing (Downsampling):
    * Safe class (Label 0) downsampled to 5000 records.
    * Attack class (Label 1) retains all 1620 context-rich records.
- Dataset Shuffling: Randomized shuffle with seed 42 applied BEFORE splitting.
- Train/Test Split: Implemented 80% Training and 20% Testing split.

---------------------------
- Total Balanced Records: 6,620
- Safe Messages (Label 0): 5,000 (75.5%)
- Attack Messages (Label 1): 1,620 (24.5%)

3. SPLIT STATISTICS
-------------------

- Testing Set (afad_test.csv): 1,324 records

4. KEY FINDINGS
---------------
- Context Filtering: The word-count filter (>=3 words) removed 2 extremely short attack messages and numerous short safe messages, ensuring the model trains on meaningful semantic patterns rather than generic stop-words or one-word signals.
- Ready for Training: The split ensures the model can be properly evaluated on unseen data, maintaining the class ratio across both sets due to shuffling.

==============================
PHASE 4: ADVANCED FEATURE ENGINEERING
==============================
Date: 2026-03-14

Step 4.1 — Define Cue Dictionaries (Core Architecture)
------------------------------------------------------
- Process: Created 'AFAD_Project/features/cue_dictionary.py' containing explicit manipulation word lists.
- Categories defined:
    * Familiarity Cues (e.g., "bro", "it's me")
    * Urgency Cues (e.g., "immediately", "asap")
    * Emotional Cues (e.g., "please help", "emergency")
    * Authority Cues (e.g., "boss", "manager")
- Result: Established a hardcoded baseline of social engineering indicators for targeted detection.
- Findings: These cues represent high-precision markers that distinguish Artificial Familiarity attacks from generic spam.

Step 4.2 — Convert Text to TF-IDF Features
------------------------------------------
- Process: Implemented 'TfidfVectorizer' in 'AFAD_Project/models/train_text_model.py' to transform raw messages into numerical vectors. 
- Configuration:
    * max_features: 5000 (Top important terms)
    * ngram_range: (1, 2) (Unigrams + Bigrams)
- Result: 
    * Training Matrix: (5296, 5000)
    * Testing Matrix: (1324, 5000)
- Findings: TF-IDF effectively captures both single words and powerful bigram patterns like "send money" and "verify account", enabling the model to recognize multi-word attack signals.

Step 4.3 — N-gram Detection (Familiarity Analysis)
-------------------------------------------------
- Significance: Single words (unigrams) are often too weak to distinguish attacks from normal communication.
- Strategy: By setting 'ngram_range=(1,2)', the system captures critical phrases and word combinations.
- Examples Captured:
    * "it's me" (impersonation)
    * "please help" (emotional manipulation)
    * "send money" (scam intent)
    * "don't tell anyone" (secrecy)
- Result: The feature space is enriched with semantic context necessary for detecting sophisticated Artificial Familiarity attacks.

Step 4.4 — Psychological Feature Counters
-----------------------------------------
- Process: Developed a custom feature extractor in 'AFAD_Project/features/custom_feature_extractor.py' that calculates structural scores based on psychology-driven cues.
- Implementation: Built 'count_cues' functions to scan messages for keywords defined in 'Step 4.1'.
- Result: 
    * Sample Input: "Hey bro urgent please help send money"
    * Output: [Familiarity: 1, Urgency: 1, Emotional: 1, Authority: 0]
- Findings: These counters provide a separate numerical dimension from TF-IDF, allowing the model to explicitly weight psychological triggers regardless of general word frequency.

Step 4.5 — Combine TF-IDF + Psychological Features
-------------------------------------------------
- Process: Merged the sparse TF-IDF matrix (linguistic signals) with the dense psychological feature counters (manipulation signals) using 'scipy.sparse.hstack'.
- Feature Vector Composition:
    * TF-IDF features: 5,000 (Capture vocabulary and N-grams)
    * Psychological features: 4 (Familiarity, Urgency, Emotional, Authority)
- Result: 
    * Final Training Feature Matrix: (5296, 5004)
    * Final Testing Feature Matrix: (1324, 5004)
- Findings: The combined vector provides a rich, multi-layered representation of messages, blending general language patterns with specific social engineering cues designed for the AFAD system.

Step 4.6 — Example Full Feature Pipeline (Architectural Flow)
------------------------------------------------------------
- Process: Mapped the end-to-end lifecycle of a single message through the AFAD system:
    1. Raw Text Input
    2. Preprocessing (Cleaning/Filtering)
    3. TF-IDF Analysis (Linguistic Signals)
    4. Cue Detection (Psychological Counters)
    5. Feature Combination (Vector Creation)
    6. ML Model Classification
- Result (Simulation):
    * Input: "Hey bro it's me urgent help"
    * Transformation: Converted to a 5004-dimensional vector.
    * Prediction: Suspicious / Attack.
    * Risk Score: ~82% probability.
- Findings: the integrated pipeline creates a robust detection mechanism that doesn't just look for words, but evaluates the 'intent' and 'influence' within the text.

Step 4.7 — Design Rationale (The 'AFAD' Advantage)
-------------------------------------------------
- Rationale: Most generic phishing detectors rely solely on linguistic frequency (TF-IDF). While effective for mass spam, they often fail against targeted social engineering.
- Innovation: The addition of explicit psychological feature counters (Familiarity, Urgency, Emotion, Authority) mimics the human decision-making process.
- Outcome: This hybrid design allows the model to prioritize "intent" over "vocabulary". It is specifically optimized to detect malicious manipulation rather than just keyword repetition.
- Project Goal Alignment: Successfully transitions the project from a "Simple Spam Filter" to an "Artificial Familiarity Attack Detector," meeting high academic and security standards.

Step 5.4 — Split Features and Labels
------------------------------------
- Process: Isolated 'message' as the independent variable (X) and 'label' as the dependent target variable (y).

Step 5.5 — Train/Test Split (Stratification)
--------------------------------------------
- Process: Implemented 'train_test_split' with 'stratify=y' on the combined 'data/final_afad_dataset.csv'.
- Configuration:
    * test_size: 0.2 (80% Training / 20% Testing)
    * random_state: 42 (Reproducibility)
    * stratify: y (Ensures class proportions are identical in train and test sets)
- Result: 
    * Training Set: 5,296 records
    * Testing Set: 1,324 records
- Findings: Stratification is critical for the AFAD project to prevent the model from becoming biased during evaluation, especially given the 3:1 class ratio. The model achieved a high accuracy of 98.94% with this setup.

Step 5.7 — Train Logistic Regression Model
------------------------------------------
- Process: Initialized 'LogisticRegression' with 'max_iter=1000' and fitted it to the 5004-dimensional training matrix (X_train_final).
- Outcome: The model successfully learned to weight linguistic and psychological signals simultaneously. 
- Learned Patterns:  
    * Attack Signal: High 'familiarity' (bro) + High 'urgency' (urgent) + scam keywords (send money) → High Attack Probability.
    * Safe Signal: Professional vocabulary (Enron-based) + low psychological cue counts → High Safe Probability.
- Result: The training phase is complete, resulting in a robust 'afad_model.pkl'.

Step 5.8 — Test the Model
------------------------
- Process: Evaluated the trained Logistic Regression model on the unseen stratified test set (1,324 records).
- Result: 
    * Final Accuracy: 98.94%
    * Precision (Attack): 99%
    * Recall (Attack): 97%
- Findings: The model performed exceptionally well, exceeding the initial project expectation of 85-92%. The combination of TF-IDF and psychological features proved to be a highly effective strategy for detecting Artificial Familiarity Attacks with minimal false positives.
